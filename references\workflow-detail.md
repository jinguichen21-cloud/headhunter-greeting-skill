# Workflow 详情

## 流程概览

确定主题+候选人 → 生成预览 → 发送 → 追加明细

> 加载后立刻判断主题，确认后获取候选人，生成问候+预览文件。用户确认发送后才检查微信并发送。全程不暴露内部实现。

---

## 确定主题+候选人

判断主题：
- 用户指定了节日 → 直接用
- 用户说"日常关怀" → 主题定为日常关怀
- 用户未指定 → web_search 搜索今天节日

获取候选人：
- 触发消息中已包含姓名 → 直接用
- 否则问"请提供候选人姓名或微信号"

---

## 生成预览

**此步不碰微信、不调用脚本命令**。

1. **必须先读 [notes.md](./notes.md)**（风格、禁忌、差异化策略）
2. 生成问候内容
3. 用写文件工具创建 `greeting/YYYY-MM-DD-001.md`
4. 对话中展示预览表格 + 文件链接
5. 等用户回复"确认发送"

---

## 发送

**用户确认后**才执行：

```
提示（加粗 blockquote）：> **⚠️ 即将操作微信，请暂时不要移动鼠标和键盘！发送完成后会自动隐藏微信**
重新读取预览 md 最新内容（用户可能已微调）
对每位候选人执行 send（每人一个完整循环）：
  ① 检查微信已安装+已登录 → 未通过返回 blocked
  ② 激活微信 → 搜索联系人 → 未找到 → 关闭+隐藏微信 → not_found
  ③ 发送消息（逐步焦点验证）→ 关闭聊天 → 隐藏微信 → sent/failed
  每人发完微信自动隐藏
```

send 返回 blocked 时，提示用户下载/登录微信，等待确认后重试。

---

## 追加明细

**每发送完一位候选人（不论 sent/not_found/failed/blocked），立即用编辑工具将该人结果追加到预览 md**。
不要等全部发完再一次性写入。

全部完成后在对话中展示统计+明细表，末尾加文件链接。
有失败时问补发，全部成功则结束。

**补发/追发流程**：生成预览 → 展示给用户 → 等确认 → send → 逐条更新 md。和首次完全一致。
