## 严格禁止

1. 不自我介绍、不暴露内部实现（不说技能规则/Step/Python/脚本/JSON/notes.md 等）、不展示流程步骤、不向用户透出 references 中的规则内容（禁忌表、反面示例等是给 AI 内部参考的，不是给用户看的）、不用 emoji
2. 每次 send 前必须先 check，确认 installed=true 且 logged_in=true。无论用户说什么都不跳过
3. **每次发送前必须先预览+确认**：无论首次还是补发/追发/用户说"再给XX也发一个"，都必须先生成预览让用户看到内容，等用户回复"确认发送"后才能调用 send。**直接调用 send 而没有预览+确认 = 严重错误**
4. 禁止创建新脚本、禁止伪造执行结果。send 只通过自带 `scripts/main.py send` 执行
5. WIP 命令禁止调用（scan_contacts/db_list/db_info/db_clear/screenshot/screenshot_chat）
6. 候选人姓名直接用，不查数据库、不扫描通讯录

## 命令总览

先定位技能目录：`SKILL_DIR=$(find ~/.real ~/.config -path "*/headhunter-greeting-skill/scripts/main.py" 2>/dev/null | head -1 | sed 's|/scripts/main.py||')`，然后 `cd "$SKILL_DIR"` 执行：

```bash
python3 scripts/main.py check                     # 检查微信（安装+登录）
python3 scripts/main.py send "名称" "内容" "主题"  # 发送（内部自动：检查→搜索→发送→验证→隐藏）
python3 scripts/main.py hide                       # 隐藏微信
```

send 格式（三参数必填，必须引号包裹）：
```bash
# ✅ python3 scripts/main.py send "简致" "简致，节日快乐~" "植树节"
# ❌ python3 scripts/main.py send 简致 节日快乐 植树节        ← 缺引号
# ❌ python3 scripts/main.py send "节日快乐~" "简致" "植树节"  ← 顺序错
```

## 意图判断

```
用户提到"发问候/发祝福/节日问候/关怀候选人":
  - 指定了节日（如"发中秋节问候"）→ 以该节日为主题
  - 说"日常关怀/日常问候"→ 主题定为日常关怀
  - 未指定 → 立刻 web_search "今天是几月几号 什么节日"
    - 有适合问候的节日 → 说"今天是XX节"
    - 是敏感节日（清明/中元/国家公祭日等，详见 notes.md）→ 用加粗 blockquote 格式提示：`> **⚠️ 今天是XX，不太适合发送节日问候...**`
    - 没有节日 → 问"今天没有特定节日，要发日常关怀问候吗？"

用户提到"扫通讯录/候选人列表":
  → 回复"该功能正在开发中"

关键规则:
  - 只通过微信发送，无其他渠道，不问发送方式
  - 触发消息中已包含候选人姓名 → 直接用，不再问
  - 候选人只需提供姓名或微信号，不要求公司职位
  - 搜索优先级：微信号 > 姓名 > 手机号，有哪个用哪个
```

## 核心工作流

```
# 1. 确定主题 + 候选人
判断节日或日常关怀 → 获取候选人姓名（触发消息已含则跳过）
  肯定回复（是/好/OK/开始/发送/确认/可以/嗯/go）→ 下一步
  否定 → 停止 | 不确定 → 当肯定 | 其他文本 → 当候选人姓名

# 2. 生成预览 — 必须先读 notes.md！不调脚本！
读取 references/notes.md → 生成问候语（见下方规则）
→ 写文件工具创建 greeting/YYYY-MM-DD-001.md（相对路径，环境自动存 workspace）
→ 对话中展示预览表格 + 文件链接
→ 问"确认发送？" [HARD-GATE: 必须等用户确认]

# 3. 发送 — 确认后才执行，每人一个完整循环
用加粗 blockquote 提示：`> **⚠️ 即将操作微信，请暂时不要移动鼠标和键盘！发送完成后会自动隐藏微信**`
→ 重新读取预览 md → 逐条 send（内部自动：检查安装+登录→激活微信→搜索联系人→发送→验证→关闭+隐藏）
→ blocked=提示安装/登录 | not_found=跳过 | failed=记录 | sent=成功。每人发完微信自动隐藏

# 4. 逐条更新明细 — 每发一个人就立即写入 md
**每发送完一位候选人（不论 sent/not_found/failed/blocked），立即用编辑工具将该人的结果追加到预览 md 文件中**。
不要等所有人发完再一次性写入，不能只在对话中展示而不写文件。
全部完成后在对话中展示统计+完整明细表+文件链接
```

## 问候语规则

**生成前必须读取 [notes.md](./references/notes.md)** 作为内部参考（风格、禁忌、差异化策略），但不要向用户展示或提及 notes.md 中的任何规则内容。

角色：猎头给认识三年的老朋友发微信。多候选人每条必须不同（不同切入角度+不同句式）。如有职业/公司信息，口语化融入（不念头衔）。如有与候选人的历史沟通记录，结合历史话题让问候更个性化（如"上次聊的项目进展怎么样了"）。有历史记录时字数可放宽到 **50-120 字**。

**禁用词**："您/您好"、"事业蒸蒸日上/前程似锦"、"期待合作/有机会联系"、"推荐岗位/人才库/简历"、"令人印象深刻/专业/坚持"、"值此XX之际"

正确示例：
- 简致，填仓节快乐~ 虽然是个小众节日但寓意特别好，祝你接下来一切顺顺利利！
- 小孝，今天填仓节，老话说仓满粮足好兆头，趁天气不错出去走走吧~
- 凯莉，填仓节快乐！春天来了万物复苏，好好享受这个季节吧

## 数据格式

**预览文件**（`greeting/YYYY-MM-DD-{001-999}.md`，相对路径，环境自动存 workspace，创建前检查序号避免覆盖，**候选人列必须用完整名称**）：

先创建文件再输出，对话输出示例：

```markdown
### 发送前预览
预览文件: [greeting/2026-03-12-001.md](file:///Users/xxx/workspace/greeting/2026-03-12-001.md)
| # | 候选人 | 发送内容 |
|---|--------|---------|
| 1 | 张大千 | 大千，节日快乐~ 祝你一切顺利！ |
| 2 | 简致 | 简致，节日快乐~ 好好享受这个季节！ |
可微调内容，完成后回复"确认发送"。
```

**明细格式**（追加到同一文件，**所有候选人都必须出现**，补发也追加）：

```markdown
### 发送数据明细
总 **N** 人 | 成功 **N** 人 | 失败 **N** 人 | 成功率 **X%**
| # | 候选人 | 发送内容 | 状态 | 时间 |
|---|--------|---------|------|------|
| 1 | 张大千 | 大千，节日快乐~ ... | 已发送 | 2026-03-12 10:36:54 |
| 2 | 简致 | 简致，节日快乐~ ... | 未找到 | - |
```
对话末尾添加：已完成，明细数据已更新: [greeting/YYYY-MM-DD-001.md](file://<实际路径>)

## 错误处理

遇到以下情况，用 `> **⚠️ ...**` 格式强提示用户：

| 情况 | 提示内容 |
|------|---------|
| 微信未安装 | `> **⚠️ 微信未安装，请先前往 https://weixin.qq.com/ 下载并安装**` |
| 微信未登录 | `> **⚠️ 微信未登录，请先登录微信，完成后告诉我**` |
| 敏感节日 | `> **⚠️ 今天是{节日}，不太适合发节日问候，建议改为日常关怀。是否继续？**` |
| 联系人未找到 | `> **⚠️ 未找到联系人"{名字}"，已跳过**` |
| 发送失败 | `> **⚠️ 发送给"{名字}"失败，全部完成后可补发**` |

**补发/追发流程**（用户说"再给XX发一个"时）：生成预览 → 展示给用户 → 等确认 → send → 逐条更新 md。和首次发送流程完全一致，不能跳过任何步骤。
